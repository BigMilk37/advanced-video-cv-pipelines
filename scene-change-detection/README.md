# Scene Change Detection (SCD)

A Python-based video analysis project implementing heuristic and machine learning approaches to detect scene transitions in video files. Developed as part of a course on intelligent video processing methods.

---

## Overview

The notebook is split into two tasks:

**Task 1 — Heuristic SCD**: Build a scene change detector using hand-crafted features and threshold logic, without high-level CV libraries.

**Task 2 — ML-based SCD**: Extend the heuristic approach with a machine learning classifier trained on extracted frame features.

Both tasks also include a **bonus subtask**: detecting dissolve/fade transitions (gradual blends between scenes), in addition to hard cuts.

## Tasks

### Task 1: Heuristic Scene Change Detector

Implements `scene_change_detector(frames)` — a from-scratch detector that:

- Downsamples and blurs frames for efficiency
- Extracts spatially-binned HSV color histograms and grayscale structural features
- Computes a combined color + ZNCC distance metric between consecutive frames
- Uses a sliding-window local peak detector to identify hard cuts
- Applies dynamic thresholding based on a rolling median noise floor
- Includes flash/strobe suppression via a look-ahead consistency check

**Bonus**: `scene_change_detector_dissolve(frames)` — detects gradual dissolve transitions by modelling each frame as a linear blend of its temporal neighbors and measuring the blend fit ratio across multiple window sizes.

### Task 2: ML-based Scene Change Detector

Implements `scene_change_detector_ml(frames)` — uses the same feature extraction pipeline as Task 1 but replaces the manual threshold with a learned classifier:

- Features are extracted per-frame (color histograms, structural thumbnails)
- An SVM classifier (trained on labelled frame pairs from training videos) predicts cut/no-cut
- The trained model is serialised to `model.pkl` via `pickle` and loaded at inference time

**Bonus**: `scene_change_detector_dissolve_ml(frames)` — ML-augmented dissolve detector.

## Evaluation

All detectors are evaluated using **F1-score**, computed over predicted scene change frame indices against ground truth. Frames marked as `trash`, `dissolve`, or `fade` are excluded from cut evaluation.

```
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
F1        = 2 * (precision * recall) / (precision + recall)
```

For dissolve detection, a prediction counts as a TP if it falls within the annotated dissolve range. Multiple predictions in the same range only count once; extras are counted as FPs.

The main metric to track is `_mean_f1_score` returned by the evaluation runners.

## Constraints

Per course requirements, the following OpenCV functions are permitted:
- Reading video: `cv2.VideoCapture`
- Color space conversion: `cv2.cvtColor`
- Sobel edge detection: `cv2.Sobel`

All other image analysis must be implemented using NumPy. Additional libraries (e.g. `sklearn`, `pandas`) require prior approval via the course chat.

## Grading

| Component | Formula |
|---|---|
| Task 1 (Base) | `20 * (max(0, X - 0.5) / (Y - 0.5))^2 + Bonus` |
| Task 2 (Bonus dissolve) | `5 * max(0, X - 0.1) / (Y - 0.1) + Bonus` |

Where `X` is your F1-score on the hidden test set and `Y` is the best submission's F1-score. Final evaluation uses a held-out video set — avoid overfitting to the training split.
